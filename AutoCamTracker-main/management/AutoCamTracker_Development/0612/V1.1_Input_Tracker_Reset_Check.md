# AutoCamTracker V1.1 Input / Tracker / Reset 檢查

日期：2026-06-12

## Input 更新檢查

- 已新增 `video_url` 作為 Input 下拉選單選項。
- 已在 `Browse Video` 正下方新增 URL 貼入口。
- 原本的影片選取文字保留在 URL 欄位下方獨立一列，所以 URL 欄位不會覆蓋或取代目前的 video selected 文字。
- 當 URL 欄位輸入網址後，按 Return 或欄位失焦時，Input 會自動切換成 `video_url`。
- URL 影片現在會先透過 `yt-dlp` 解析網頁影片網址，例如 YouTube watch URL，再把解析出的實際串流網址交給 OpenCV `VideoCapture` 播放。
- 直接影片或串流網址仍支援 `http`、`https`、`rtsp`、`rtmp`。
- 實測：`https://www.youtube.com/watch?v=jNQXAC9IVRw` 可解析成 `googlevideo.com` 串流，OpenCV 可成功開啟並讀取首幀。
- 實測：`P8Uh9f0EaU8` 與 `XiNvTSlu99Q` 兩個 YouTube ID 都可解析成 `googlevideo.com` 串流，OpenCV 可成功讀取首幀，frame shape 為 `(360, 640, 3)`。

## Screen Region 選取檢查

- 再次按下 `Screen Region` 時，上一個螢幕選取範圍會立即清除。
- 如果目前 detector 正在使用上一個 `screen_region`，會在重新框選前先關閉。
- 開始重新框選時，runtime tracking 狀態會重置。
- 新拖曳出的範圍會成為下一次使用的 active `screen_region`。

## Before / After 顯示比例檢查

- 修正前：UI 會把來源畫面硬塞進 View 區塊目前的寬高，因此可能把 16:9 影片壓成很扁的比例。
- 修正後：Before 與 After 顯示都會依照來源 frame 的原始比例等比縮放。
- Reframer 的輸出尺寸會同步成來源 frame 的原始寬高，例如 1920x1080，不再使用 View 面板的任意顯示寬高當作輸出尺寸。
- `Apply Size` 手動套用時也會依來源比例修正尺寸，避免使用者輸入任意寬高後造成畫面變形。
- 實測：1920x1080 測試 frame 經 Reframer 輸出仍為 1920x1080，crop window 為 `(0, 0, 1920, 1080)`。

## Tracker 功能檢查

`Tracking > Tracker` 下拉選單目前有實際作用，不是裝飾性的 UI。

- `botsort`
  - UI 值：`self.tracker_var = "botsort"`。
  - 會傳入 `InputConfig.tracker_name`。
  - `VideoDetector.track_frame()` 會使用 `self.model.track(..., tracker="botsort.yaml", persist=True, ...)`。
  - Track ID 來源是 Ultralytics YOLO tracking mode。

- `deepocsort`
  - UI 值：`self.tracker_var = "deepocsort"`。
  - 會傳入 `InputConfig.tracker_name`。
  - `VideoDetector.load_model()` 會建立 `DeepOcSortAdapter`。
  - `VideoDetector.track_frame()` 會先用 YOLO `predict()` 偵測，再把車輛 detections 丟進 `DeepOcSortAdapter.update()`。
  - Track ID 來源是專案內 bundled 的 `code/model/Deep-OC-SORT-main` tracker adapter。

結論：Tracker 下拉選單會改變 tracking 執行路徑與 track ID 來源。

## Reset 按鈕檢查

目前 `Tracking > Reset` 按鈕會呼叫 `reset_tracking()`，再呼叫 `_reset_runtime_state()`。

目前 Reset 會做的事：

- 使用 `self.store.reset()` 清除偵測歷史。
- 使用 `self.target_tracker.clear_selection()` 清除目前選取的目標車輛。
- 使用 `self.reframer.reset()` 重置 reframe / crop 狀態。
- 清除 `last_frame_shape`。
- 將 skipped frame count 重置為 `0`。

目前 Reset 不會做的事：

- 不會停止 camera / video / screen source。
- 不會關閉或重新載入 YOLO model。
- 不會直接重置底層 tracker object 的內部狀態。
- 不會重置播放位置或來源選擇。

結論：Reset 目前代表「清除 app 層的 tracking / reframing 狀態」，不是「重新啟動整個 input / model pipeline」。
