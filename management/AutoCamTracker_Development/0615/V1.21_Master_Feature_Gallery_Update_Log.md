# V1.3 Master Feature Gallery Update Log

## 核心變更

- 停用 V1.2 observation embedding gallery。
- `VehicleIdentityStore` 改為 metadata-only Identity DB：
  - 保存 GID、display name、class、last local track、last frame、confidence、last bbox、center、metadata。
  - 不再每幀保存 embedding observation。
  - schema migration 會移除 legacy `observations` table。
- 新增 `FeatureGallery`：
  - 支援 Master / Pending / Candidate gallery type。
  - Master 是唯一正式 ReID 身份來源。
  - 每台車 Master feature 上限 500 張。
  - 新增 crop quality filter。
  - 新增 duplicate feature rejection。
  - 新增 Master Gallery Top-K matching。
  - 預留 FAISS / Qdrant / Milvus backend 介面，預設仍使用 SQLite。

## 手動入口

- `Add Feature`
  - 選 GID 後，使用目前可見 bbox 抽 ReID feature。
  - 通過 quality 與 duplicate checks 後才寫入 Master。
- `Find GID`
  - 選 GID 後，手動使用 Master Gallery 對目前 detections 做 Top-K matching。
- `Link BBox`
  - 選 GID 後，點 Before 畫面的 bbox，把 bbox 手動綁到同一台車。
  - 用於 feature 不足、跨攝影機補資料前的人工校正。

## 停用的自動行為

- 點 bbox 不抽 ReID feature。
- 每幀追蹤不抽 ReID feature。
- UI refresh 不抽 ReID feature。
- Identity DB 不再自動 matching 到舊 observation embeddings。
- `botsort_reid` tracker config 已移除；正式 tracker 選項回到普通 `botsort` 與 `deepocsort`。
