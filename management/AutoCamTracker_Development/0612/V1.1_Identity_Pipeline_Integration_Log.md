# V1.1 Identity / Pipeline Integration Log

日期：2026-06-12

## 本次導入

- 保留現有 V1.1 UI、URL、timeline、playback speed、Deep OC-SORT / BoT-SORT 選項。
- 新增 `FrameData` 作為每一幀的結構化輸出資料。
- 新增 `PipelineProcessor`，把 per-frame pipeline 從 Tkinter UI 中抽出：
  - detections
  - scene cut
  - detection store
  - global identity update
  - reframe
  - FrameData assembly
- 新增 `GlobalIdentityManager`。
- 新增 `ReacquireEngine`。
- 新增 `SceneCutDetector`。
- Deep OC-SORT adapter 新增 reset capability，讓 seek / camera cut 可重置本地 tracker。

## 行為變更

- 使用者點選 bbox 或 Auto Track 時，現在會建立 `global_vehicle_id`。
- UI 狀態列會顯示：
  - tracking status
  - global id
  - local tracker id
  - lost frame count
  - reacquire score
  - camera cut flag
- local tracker id 消失時，不會立刻清掉選取車輛。
- camera cut 發生時會重置本地 tracker state，但保留 selected global identity。
