# V1.2 ReID Gallery Update Log

日期：2026-06-12

## 本次導入

- 新增 Identity DB UI：
  - 顯示 GID、車種、local tracker id、觀測次數、重複寫入次數、最後 frame、confidence。
  - 點選 DB row 可用資料庫身份反向比對目前畫面中的車輛。

  - 使用 Ultralytics ReID encoder。
  - 預設模型為 `yolo26n-reid.onnx`。
  - ReID 模型不可用時會降級回原本 histogram matching。
- 新增 `botsort_reid` tracker 選項：
  - `code/V1/trackers/botsort_reid.yaml`
  - 開啟 `with_reid: True`
  - 使用 `yolo26n-reid.onnx`
- 更新影片視窗行為：
  - 移除手動 width / height 控制。
  - Before / After 依 Tkinter 視窗可用區域等比縮放。

## 問題確認

使用 Gmotor 影片樣本：

- 影片來源：`https://www.youtube.com/watch?v=QOjUZO56Tp8`
- 測試樣本：`outputs/gmotor_sample_0000_0200.mp4`
- 診斷標註：`outputs/gmotor_diag_same_track_0000_0015.mp4`

舊邏輯使用單張 histogram / 單張 embedding 時，同一 local track 內的資料庫分數不穩：

```text
same local track, single feature:
median: 0.506
score >= 0.72: 8.9%
```

這確認了使用者觀察到的問題：同一台車在重新出現或換鏡頭後，很難被現有 DB matching 找回。

## V1.2 改善

改成 gallery matching：同一個 GID 會保存多筆 observation embedding，查詢時與最近 80 筆 observation embedding 比對，取最佳分數。

```text
same local track, accumulated gallery:
median: 0.922
score >= 0.72: 100%
score >= 0.82: 95.2%
```

## 目前限制

- ReID gallery 需要先累積多筆高品質 observations；只用單張小 crop 仍不穩。
- 目前仍是互動式重新辨識，不是完全自動跨鏡頭 MTMC pipeline。
- 賽道轉播中車輛很小、角度變化大、遮擋多，後續仍需要 tracklet-level selection 與更好的 crop quality filter。
- `botsort_reid` 第一次啟動可能需要下載 `yolo26n-reid.onnx`，模型檔不納入 git。
